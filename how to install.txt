명령어
./gradlew bootRun : 애플리케이션 실행
./gradlew clean build : 클린 빌드 및 재동기화

------------------------------------------------------------------------------------------------------------------------------------------------------------
build.gradle 파일을 수정한 후에는 매번 ./gradlew clean build를 수행할 필요는 없습니다. Gradle은 의존성이나 빌드 설정이 변경될 때만 해당 작업을 필요로 하며,
bootRun 작업에 작은 설정을 추가하는 정도는 별도의 클린 빌드가 필요하지 않습니다.

정리
작업 수정 (예: bootRun에 브라우저 열기 코드 추가): clean build 없이도 바로 적용됩니다.
의존성 추가/변경 (예: 새로운 라이브러리 추가): build.gradle에서 의존성을 추가하거나 제거한 경우에는 클린 빌드를 권장합니다.
리소스나 컴파일 대상 변경: 리소스나 주요 컴파일 대상이 변경되면, build만으로도 충분히 변경 사항이 반영됩니다.
따라서 일반적으로 **작업 내 로직 수정 후에는 ./gradlew bootRun**을 바로 실행해도 무방합니다. 필요한 경우에만 clean을 함께 사용하는 것이 효율적입니다.
